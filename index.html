Two steps to streaming

POST /video/ingest

{
  "srcVideo": "Puppies_Bath_4096x2304_375mb",
  "archiveSource": true,
  "frameCapture": true
}

API Response Body

{
  "status": "Ingest",
  "guid": "4259211e-78f2-4323-9aaa-2265c27937e4",
  "srcVideo": "Puppies_Bath_4096x2304_375mb"
}

Step 1 Post

This step is an secured system call. The request body contains the file name in the S3 bucket associated with the business user, and returns the state of the task and the video’s unique ID.

The task only triggers when the reference file exists in the S3 bucket.

GET /video//?guid=4259211e-78f2-4323-9aaa-2265c27937e4

API Response Body

{
  "url": "https://d1kx6xd26ja9y4.cloudfront.net/4259211e-78f2-4323-9aaa-2265c27937e4/hls/Puppies_Bath_4096x2304_375mb.m3u8"
}

Step 2 Stream

This is a developer call. After the ingestion process is complete, the files are uploaded to a CDN, and at the moment we serve 2 formats for video streaming, HLS & DASH. We allow for the developer to pick the format.
